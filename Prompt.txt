2. Ligar à base de dados e executar a query
Logo após este bloco existente:

3. Adicionar o seguinte bloco PHP (bloco try/catch) para ligação e execução da query: 
try {
 $ligacao = new PDO(
 "mysql:host=" . MYSQL_HOST . ";dbname=" . MYSQL_DATABASE . ";charset=utf8",
 MYSQL_USERNAME,
 MYSQL_PASSWORD
 );
 $ligacao->setAttribute(PDO::ATTR_ERRMODE, PDO::ERRMODE_EXCEPTION);
 $resultados = $ligacao->query("SELECT * FROM clientes")->fetchAll(PDO::FETCH_OBJ);
 $erro = '';
} catch (PDOException $err) {
 $erro = "Aconteceu um erro na ligação.";
 $resultados = [];
}
// Fecha a ligação
$ligacao = null;


Ligação à base de dados com PDO:
 Utiliza o objeto PDO (PHP Data Objects), que oferece uma forma segura, orientada a objetos e flexível
de interagir com diferentes tipos de bases de dados.
 A string de ligação (DSN) é construída com as constantes previamente definidas (MYSQL_HOST,
MYSQL_DATABASE, etc.).
 Define-se o charset=utf8 para garantir a correta codificação de caracteres, especialmente útil para textos
com acentos ou símbolos especiais
Definição do modo de erro:
 A linha setAttribute(PDO::ATTR_ERRMODE, PDO::ERRMODE_EXCEPTION) configura o PDO
para lançar exceções (PDOException) sempre que ocorrer um erro.
 Isto permite que os erros sejam capturados no bloco catch, facilitando a deteção e tratamento controlado
de falhas.
Execução da query SQL:
 A instrução query("SELECT * FROM clientes") executa uma consulta SQL que seleciona todos os
registos da tabela clientes.
 Os resultados são obtidos com fetchAll(PDO::FETCH_OBJ) e armazenados como um array de objetos
stdClass, onde cada campo da tabela é representado como uma propriedade do objeto.
Tratamento de erros com catch:
 Se ocorrer um erro na ligação ou na execução da query, o bloco catch captura a exceção.
 É definida uma variável $erro com uma mensagem genérica, evitando que a aplicação quebre ou mostre
erros técnicos ao utilizador.
 A variável $resultados é definida como um array vazio, o que evita erros ao tentar percorrer dados
inexistentes no frontend.
Fecho da ligação:
 A linha $ligacao = null; liberta explicitamente a ligação à base de dados.
 Esta é uma boa prática, pois garante que os recursos são libertados corretamente após a sua utilização.
